# 官方大纲、教材与资料索引

## 使用顺序

建议按“大纲定边界 → 教程建体系 → 官方试题解析校准 → 自己的错题与论文素材闭环”的顺序准备。资料多并不会自动提高通过率，最重要的是把同一知识点在选择题、案例题和论文中的三种表达连起来。

## 官方考试能力要求的学习化拆解

官方考试说明列出十项能力要求，本仓库将其转化为以下学习任务：

| 官方能力域 | 应达到的可检验结果 |
| --- | --- |
| 计算机软硬件与网络 | 能计算基本性能指标，解释存储、并发、网络与部署瓶颈 |
| 信息系统开发过程 | 能从需求、设计、实现、测试、部署和演化建立追踪链 |
| 标准与规范 | 能识别标准、知识产权、合同和合规题的适用边界 |
| 中间件与应用服务器 | 能解释消息、事务、缓存、服务治理、集群与容错机制 |
| 建模与架构设计 | 能从质量属性场景推导架构决策，并用合适视图表达 |
| 信息安全 | 能做威胁分析，给出身份、权限、数据、通信、审计和恢复控制 |
| 法律法规 | 能区分著作权、专利、商业秘密、网络与数据合规的基本问题 |
| 行业适配 | 能把领域约束、业务峰谷、数据敏感性与架构方案对应起来 |
| 应用数学 | 能处理可靠性、可用性、性能、容量、概率与简单决策计算 |
| 专业英语 | 能识别架构、质量属性、网络、安全和软件工程常用术语 |

## 合法补充资料

可补充以下公开、合法资料，但不要让它们取代考试大纲：

- ISO/IEC/IEEE 42010 软件、系统与企业架构描述概念；
- OWASP Top 10、OWASP ASVS 的公开版本；
- 云厂商公开的架构良好实践框架，用于理解可靠性、安全、性能和成本权衡；
- 主流数据库、消息队列、容器编排与可观测性项目的官方文档；
- 自己参与项目的需求、架构决策记录、故障复盘和性能报告（须脱敏）。

## 不建议收集

- 盗版教材 PDF、网盘扫描件或未经授权的课程录屏；
- 来源不明的“押题”“保过题库”和所谓考场回忆完整版；
- 只给结论、不写适用条件和权衡的万能论文模板；
- 版本不明、与现行第 2 版大纲不对应的章节笔记。
