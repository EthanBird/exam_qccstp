# 案例分析方法

## 核心原则

案例题不是“看到微服务就写注册中心、配置中心”。高质量答案必须体现：题目场景里的约束是什么，方案为什么适用，代价如何控制，结果怎么验证。

通用回答链：

> 问题/质量属性 → 题干证据 → 架构决策 → 工作机制 → 风险与权衡 → 验证指标

## 五步作答法

1. **圈定题型**：架构风格、质量属性、评估、数据库、分布式、安全、可靠性或演化。
2. **提取约束**：用户量、峰值、延迟、可用性、数据一致性、合规、存量系统、预算和团队。
3. **按问拆分**：题目问三点就分三点回答，编号与题号保持一致。
4. **用机制解释**：不能只写“使用缓存”，还要写缓存对象、键、更新/失效、一致性和降级。
5. **做闭环**：补充指标、测试或演练方式，证明方案满足要求。

## 质量属性场景模板

完整质量属性场景包括六个要素：

| 要素 | 问题 | 示例 |
| --- | --- | --- |
| 刺激源 | 谁触发？ | 已登录用户、外部攻击者、节点故障 |
| 刺激 | 发生什么？ | 突发请求、非法访问、数据库不可用 |
| 环境 | 在什么状态？ | 正常运行、峰值、发布期间、灾备切换时 |
| 制品 | 影响什么？ | API、订单服务、数据存储、部署单元 |
| 响应 | 系统怎么做？ | 限流、隔离、故障转移、拒绝并审计 |
| 响应度量 | 怎样算达标？ | P95 延迟、错误率、RTO、数据丢失量 |

答题时优先把“快、稳定、安全、易改”改写为可测量描述。

## 高频架构风格对比

| 风格 | 适合 | 主要收益 | 典型代价/风险 |
| --- | --- | --- | --- |
| 分层 | 职责稳定、需要隔离变化 | 易理解、替换、测试 | 跨层调用开销、层次穿透 |
| 管道—过滤器 | 连续转换、编译/ETL | 复用、并行、组合 | 数据格式转换、交互式处理不佳 |
| 仓库 | 多组件共享一致数据 | 集中管理、数据一致 | 单点、瓶颈、模式耦合 |
| 事件驱动 | 异步解耦、状态传播 | 弹性、扩展、削峰 | 顺序、重复、追踪和最终一致性 |
| SOA | 企业异构集成、复用服务 | 标准化、互操作 | 治理复杂、中心化基础设施风险 |
| 微服务 | 独立演进和部署、团队自治 | 独立扩缩、故障隔离 | 分布式复杂度、一致性、运维成本 |
| 微内核 | 稳定核心 + 可变插件 | 扩展、隔离可选能力 | 插件协议、版本兼容、核心边界设计 |

## 分布式系统答题清单

涉及远程调用时，至少检查：

- 是否设置连接、读取和总超时；
- 重试是否只用于可重试错误，是否有退避和抖动；
- 操作是否幂等，幂等键保存在哪里、有效期多久；
- 是否需要熔断、隔离、限流和降级；
- 消息是否可能重复、乱序、丢失，消费者如何去重；
- 数据一致性目标是强一致还是最终一致；
- 是否具备日志、指标、追踪和关联 ID；
- 失败后如何补偿、回放、人工处理和审计。

## 数据与缓存答题清单

- 访问模式：读写比例、热点、范围查询、排序、聚合；
- 模型与索引：主键、复合索引顺序、覆盖索引、分区键；
- 事务：隔离级别、锁范围、长事务和死锁；
- 扩展：复制、读写分离、分片、跨分片查询和全局 ID；
- 缓存：Cache Aside/读写穿透/写回；TTL、失效、双删或版本；
- 防护：穿透、击穿、雪崩、热点键和大 Key；
- 恢复：备份、PITR、复制延迟、RPO/RTO 和恢复演练。

## 安全题答题框架

按“资产—威胁—入口—控制—证据”组织：

1. 识别账号、密钥、接口、业务数据、日志和供应链资产；
2. 分析伪造、篡改、抵赖、信息泄露、拒绝服务、权限提升等威胁；
3. 在身份、权限、输入、通信、数据、运行、审计和恢复各层布置控制；
4. 明确密钥轮换、最小权限、职责分离和默认拒绝；
5. 用安全测试、审计日志、告警、演练和整改闭环证明有效。

## 常用计算

### 可用性

\[
A = \frac{MTBF}{MTBF + MTTR}
\]

降低 MTTR 往往和提高 MTBF 同样重要：自动检测、故障切换、回滚和演练都是缩短恢复时间的手段。

### 指数可靠性模型

\[
R(t)=e^{-\lambda t}
\]

独立串联系统可靠性为各部件可靠性的乘积；独立并联系统可靠性为：

\[
R_{parallel}=1-\prod_i(1-R_i)
\]

### Little 定律

\[
L=\lambda W
\]

系统平均在途请求数等于平均到达率乘以平均停留时间，可用于并发与容量估算；应用前要确认系统处于相对稳定状态且统计口径一致。

## 常见失分

- 复述题干，没有给方案；
- 堆产品名，没有解释机制和边界；
- 只谈收益，不谈一致性、复杂度、成本和失败路径；
- 题目要求两点却写一大段，评分点难以识别；
- 不引用题干证据，答案像任何系统都能套用；
- 给出不可测的目标，例如“显著提升”“基本无故障”。

## 训练方法

1. 第一次按考试时间独立作答；
2. 对照合法取得的官方解析，标出缺少的关键词；
3. 给每个丢分点标记知识、场景、计算、时间或表达原因；
4. 24 小时后不看答案重写；
5. 一周后只看题干口述决策链，验证是否真正掌握。
