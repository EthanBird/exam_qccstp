# 架构速查表

本页用于最后四周快速复盘。它不能替代大纲和完整学习。

## 质量属性战术

| 属性 | 常见战术 | 需要追问的代价 |
| --- | --- | --- |
| 性能 | 缓存、并发、批处理、连接池、分区、负载均衡、减少往返 | 一致性、内存、热点、复杂度、尾延迟 |
| 可用性 | 冗余、心跳、故障转移、超时、重试、熔断、隔离、降级 | 成本、脑裂、重复请求、容量预留 |
| 可修改性 | 抽象、封装、限制依赖、插件、配置化、稳定接口 | 间接层、调试难度、过度设计 |
| 安全性 | 认证、授权、最小权限、加密、审计、隔离、速率限制 | 性能、可用性、运维负担、密钥治理 |
| 可测试性 | 依赖注入、可控接口、可观测状态、测试替身、确定性 | 额外接口、环境维护、模拟偏差 |
| 可部署性 | 小批变更、自动化、灰度、蓝绿、功能开关、快速回滚 | 双环境成本、状态兼容、开关债务 |
| 可观测性 | 日志、指标、追踪、关联 ID、业务事件、健康探针 | 存储成本、采样偏差、敏感数据泄露 |

## 一致性与事务

- **强一致**：读到最新已提交结果；通常牺牲延迟或可用性。
- **最终一致**：没有新更新时，各副本最终收敛；必须设计冲突、补偿和可观测性。
- **两阶段提交**：原子性强，但协调者、锁持有和阻塞会影响可用性与性能。
- **Saga**：长事务拆为本地事务和补偿；补偿不等于数据库回滚，必须处理幂等与人工兜底。
- **事务消息/Outbox**：把业务变更与待发送事件放入本地事务，再可靠发布；要处理重复发布。
- **TCC**：Try/Confirm/Cancel 明确预留资源；业务侵入和异常处理复杂。

## 远程调用防护

推荐顺序：

1. 明确超时预算；
2. 识别是否可安全重试；
3. 设置有限次数、指数退避和抖动；
4. 用幂等键处理重复；
5. 熔断持续失败的依赖；
6. 隔离线程/连接/队列资源；
7. 限流和负载削减；
8. 降级或返回可接受的陈旧数据；
9. 记录指标、追踪和审计；
10. 通过故障注入验证。

## 缓存模式

| 模式 | 读 | 写 | 风险 |
| --- | --- | --- | --- |
| Cache Aside | 应用先查缓存，未命中查库并回填 | 更新数据库后使缓存失效 | 短暂不一致、击穿 |
| Read Through | 缓存层负责加载 | 通常配合其他写模式 | 缓存层复杂、供应商绑定 |
| Write Through | 正常读缓存 | 同步写缓存与后端 | 写延迟、后端压力 |
| Write Back | 正常读缓存 | 先写缓存，异步落库 | 数据丢失、顺序和恢复复杂 |

防护关键词：空值/布隆过滤器防穿透，互斥或逻辑过期防击穿，随机 TTL 与分批预热防雪崩，热点 Key 拆分与本地缓存缓解集中压力。

## 数据库隔离与异常

从弱到强通常为：读未提交、读已提交、可重复读、串行化。隔离越强并不总是越好，应结合并发、锁/MVCC 实现和业务不变量。常见现象包括脏读、不可重复读、幻读、丢失更新和写偏差。

## 架构评估方法

| 方法 | 重点 | 典型产出 |
| --- | --- | --- |
| SAAM | 可修改性和场景影响分析 | 直接/间接场景、受影响组件、修改成本 |
| ATAM | 多质量属性及其权衡 | 架构方法、质量属性效用树、风险、非风险、敏感点、权衡点 |
| CBAM | 在 ATAM 基础上考虑成本与收益 | 架构策略的效用、成本、风险和优先级 |

## 可靠性与灾备

- `MTBF`：平均故障间隔；`MTTR`：平均修复时间。
- `Availability = MTBF / (MTBF + MTTR)`。
- `RTO`：允许的恢复时长；`RPO`：允许的数据恢复点损失窗口。
- 备份不等于容灾，完成备份也不等于可恢复；必须做还原和切换演练。
- 多副本不等于高可用；还要考虑共同故障域、容量、选主、依赖和操作失误。

## 性能分析

- 先定义负载模型和 SLI：吞吐、P50/P95/P99、错误率、资源利用率。
- 区分平均值和尾延迟；检查排队、锁竞争、GC、I/O、网络和下游依赖。
- Amdahl 定律提醒：只优化局部受限于不可优化部分。
- Little 定律：`L = λW`，用一致口径估算平均并发。
- 压测必须说明数据规模、缓存冷热、持续时间、扩容策略和瓶颈观察。

## 安全分层

1. 身份：MFA、凭证生命周期、服务身份；
2. 权限：最小权限、RBAC/ABAC、职责分离；
3. 接口：输入验证、输出编码、防重放、限流；
4. 通信：TLS、证书、网络分段；
5. 数据：分类分级、加密、脱敏、备份、删除；
6. 运行：补丁、镜像、主机/容器加固、供应链；
7. 监控：审计、异常检测、告警、响应；
8. 恢复：密钥吊销、隔离、取证、恢复演练。

## 论文决策句式

> 由于【场景约束】，若采用【备选方案 A】会导致【风险】。我负责的架构组选择【方案 B】，通过【结构与机制】满足【质量属性及度量】；代价是【权衡】，因此又采用【控制措施】。最终通过【测试/监控/演练】验证【结果】。

这个句式只用于检查逻辑，不应原样重复充斥全文。
