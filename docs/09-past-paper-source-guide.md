# 真题数据源使用指南

这份指南说明如何使用 [`past-paper-source-catalog.csv`](../data/past-paper-source-catalog.csv) 和 [`past-paper-topic-index.csv`](../data/past-paper-topic-index.csv)，以及哪些资料适合入库。

## 1. 本仓库保存什么

- 官方考试制度、大纲和书目信息；
- 公开页面的 URL、标题、发布时间线索和访问状态；
- 题目的年份、批次、科目、主题标签和可信度；
- 基于主题元数据形成的原创趋势分析；
- 自己的练习答案、错因和复盘记录。

## 2. 本仓库不保存什么

- 盗版教材、付费课程录屏、网盘搬运包；
- 未经授权的整套试卷扫描件或大段题干、答案；
- 绕过登录、付费墙或下载限制取得的文件；
- 无法确定年份、科目或来源的“押题包”。

公开可访问不等于可自由再分发。即使来源页面展示了回忆题，仓库也只保存主题级元数据和链接。

## 3. 推荐取数顺序

1. **官方考试机构**：确认考试结构、时间、批次和大纲；
2. **官方试题解析图书**：获取完整、稳定的历史题与解析；
3. **两处独立考生回忆**：确认近年主题是否真实出现；
4. **单一培训站或社区帖子**：只当线索，标为 C 级；
5. **网盘和付费搬运站**：不下载、不入库，可直接忽略。

## 4. 字段说明

### 来源表

- `source_class`：official、training_site、candidate_recall 等；
- `confidence`：A/B/C 证据等级；
- `access_status`：public、index_or_login 等；
- `analysis_use`：该来源实际支持的结论；
- `copyright_handling`：本仓库如何避免再分发受保护内容。

### 主题表

- `normalized_topic`：只保留可用于分析的主题级转述；
- `domain`：归一化后的一级知识域；
- `tags`：用 `|` 分隔的多标签；
- `completeness`：`substantive` 表示主题信息较完整，`partial` 表示缺题或待交叉核验；
- `notes`：转录问题、批次差异或其他限制。

## 5. 更新流程

新增一批真题回忆时：

1. 先在来源表新增一行，不要立即复制题干；
2. 检查页面是否明确写出考试年份、批次和科目；
3. 把每个可核验主题写入主题表，并标注 `partial` 或 `substantive`；
4. 搜索第二个独立来源，交叉确认后才考虑把 C 调为 B；
5. 重新阅读趋势报告，只有证据足以改变结论时才更新正文；
6. 在提交信息中写明新增的批次和来源数量。

## 6. 合法获取完整题目的入口

- [中国计算机技术职业资格网考试用书](https://www.ruankao.org.cn/book/main.html)：可查新版大纲、教程和《系统架构设计师 2018 至 2022 年试题分析与解答》；
- [中国计算机技术职业资格网](https://www.ruankao.org.cn/)：报名、机考模拟、官方通知；
- 培训站公开题库：按其使用条款在线练习，不把整页内容搬回仓库。

若只是做趋势分析，主题元数据已经足够；若要做逐题训练，优先使用正版官方解析书或获得授权的题库。
